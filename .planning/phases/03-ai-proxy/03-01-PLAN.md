<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/ai/chat | Untrusted user input (messages array content) crosses into server |
| server -> MiMo API | Server sends user content to third-party AI service |
| .env -> server memory | Secret (MIMO_API_KEY) loaded into process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | routes/ai.py | mitigate | Never log/return MIMO_API_KEY value; error messages use generic text per D-08/D-09 |
| T-03-02 | Spoofing | routes/ai.py request body | mitigate | Validate request body with Pydantic model; reject malformed input with 422 |
| T-03-03 | Tampering | messages[].role=system | mitigate | Strip all role=system messages from user input per D-06; backend injects its own system prompt |
| T-03-04 | Denial of Service | /api/ai/chat | accept | No rate limiting in v1 (campus-internal, low user count); httpx timeout=30s prevents hanging |
| T-03-05 | Information Disclosure | error responses | mitigate | Catch all exceptions; return generic "AI 服务暂时不可用" without stack trace or MiMo error details |
| T-03-06 | Elevation of Privilege | system prompt injection | mitigate | User cannot override system prompt (D-06 strips it); prompt content is server-controlled |
</threat_model>
