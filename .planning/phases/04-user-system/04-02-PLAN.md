## Phase Goal

**As a** 南大学生, **I want to** 通过统一身份认证登录应用, **so that** 我可以安全地使用所有功能.

<objective>
为所有现有 API 端点添加 JWT 认证保护，前端添加认证守卫确保未登录用户跳转 CAS。

Purpose: 实现 D-08（全局认证）和 D-09（前端无 JWT 跳 CAS），让应用成为需要登录才能使用的状态。
Output: 5 个后端路由文件加认证依赖 + 前端 AuthGuard 组件包裹所有路由。
</objective>

### Task 2: 前端认证守卫 — 无 JWT 跳转 CAS

<read_first>
- what_to_eat_today_web/frontend/src/App.tsx (当前路由结构)
- what_to_eat_today_web/frontend/src/services/authService.ts (isTokenValid, redirectToCAS 函数)
</read_first>

<action>
1. 新建 `what_to_eat_today_web/frontend/src/components/AuthGuard.tsx`（per D-09）：
   - Props: `children: React.ReactNode`
   - 组件逻辑：调用 isTokenValid()，若返回 false 则调用 redirectToCAS() 并 return null（不渲染子组件）
   - 若 token 有效则 return children
   - 使用 useEffect + state 模式避免 SSR 问题：初始 state 为 "checking"，useEffect 中检查 token 后设为 "valid" 或 "invalid"
   - "checking" 状态可显示空白或简单加载提示
   - "invalid" 状态触发 redirectToCAS()

2. 修改 `what_to_eat_today_web/frontend/src/App.tsx`：
   - import AuthGuard
   - 将现有路由用 AuthGuard 包裹：`<AuthGuard><Routes>...</Routes></AuthGuard>`
   - 但 `/auth/callback` 路由必须在 AuthGuard 外可达：将 AuthCallback 路由提取到 AuthGuard 外，或在 AuthGuard 内判断当前路径为 /auth/callback 时跳过检查
   - 推荐方案：App 组件顶层用 Routes，`/auth/callback` 走独立 Route 不包裹 AuthGuard，其他路由包裹在一个需要认证的布局组件中

具体结构：
```
<Routes>
  <Route path="/auth/callback" element={<AuthCallback />} />
  <Route path="/*" element={<AuthGuard><ProtectedRoutes /></AuthGuard>} />
</Routes>
```
其中 ProtectedRoutes 是内嵌的 Routes 包含 /, /search, /ai 三个路由。
</action>

<acceptance_criteria>
- AuthGuard.tsx 存在且 export default 一个 React 组件
- AuthGuard.tsx 中引用 isTokenValid 和 redirectToCAS from authService
- App.tsx 中 /auth/callback 路由不被 AuthGuard 包裹
- App.tsx 中 /, /search, /ai 路由被 AuthGuard 包裹
- 无 localStorage jwt_token 时访问 / 触发 redirectToCAS（浏览器跳转到 authserver.nju.edu.cn）
- 有有效 jwt_token 时 / 正常渲染 HomePage
- `cd what_to_eat_today_web/frontend && npx tsc --noEmit` 无新增类型错误
</acceptance_criteria>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 前端 AuthGuard | 客户端检查可被绕过，后端认证为真正保护 |
| 后端 Depends(get_current_user) | 每个请求验证 JWT 签名和过期 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Spoofing | 绕过前端 AuthGuard | mitigate | 后端每个端点都有独立 JWT 验证，前端守卫仅为 UX |
| T-04-07 | Tampering | 修改 localStorage token | mitigate | 后端 jwt.decode 验证签名，伪造 token 无法通过 |
| T-04-08 | Denial of Service | 大量无效 token 请求 | accept | 校园内网环境，JWT 验证本身轻量 |
</threat_model>
