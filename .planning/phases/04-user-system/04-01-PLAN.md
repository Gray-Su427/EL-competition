## Phase Goal

**As a** 南大学生, **I want to** 通过统一身份认证登录应用, **so that** 我可以安全地使用所有功能.

<objective>
实现完整的 CAS 登录 + JWT 认证链路：用户通过南大 CAS 登录后获取 JWT，后续请求携带 Token 访问 API。

Purpose: 建立认证基础设施，使所有后续写入功能（评价、收藏）有用户身份支撑。
Output: 可工作的登录流程——前端跳 CAS → 回调拿 ticket → 后端验证换 JWT → 存 localStorage。
</objective>

   - `LoginRequest(BaseModel)`: 字段 `ticket: str`

   - 从 os.environ 读取 `JWT_SECRET`，缺失时 raise RuntimeError
   - 常量 `JWT_ALGORITHM = "HS256"`, `JWT_EXPIRE_DAYS = 7`
   - 常量 `CAS_SERVER = "https://authserver.nju.edu.cn/authserver"`, `CAS_VALIDATE_URL = f"{CAS_SERVER}/serviceValidate"`, `CAS_NAMESPACE = "http://www.yale.edu/tp/cas"`

6. 新建 `what_to_eat_today_web/backend/routes/auth.py`：
   - APIRouter(prefix="/api/auth", tags=["auth"])
   - POST `/login` 接收 LoginRequest body：调 validate_cas_ticket → 查询 User(student_id=uid) → 不存在则创建（created_at=datetime.utcnow().isoformat()） → 调 create_access_token → 返回 LoginResponse
   - GET `/me` 依赖 get_current_user：从 token payload 的 sub 查 User 返回 UserOut

7. 在 main.py 注册 auth router：`from routes.auth import router as auth_router` 并 `app.include_router(auth_router)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 前端 → 后端 /api/auth/login | ticket 来自 URL 参数，可被伪造 |
| 后端 → CAS 服务器 | ticket 验证请求，信任 CAS 响应 |
| 前端 localStorage → 后端 | JWT 可被窃取（XSS），但本项目为校园工具风险可接受 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | /api/auth/login | mitigate | CAS ticket 一次性验证，后端直接调 CAS 服务器确认 |
| T-04-02 | Tampering | JWT payload | mitigate | HS256 签名验证，secret 存 .env 不暴露 |
| T-04-03 | Information Disclosure | JWT_SECRET | mitigate | .env 管理，.gitignore 排除，启动缺失则 raise |
| T-04-04 | Elevation of Privilege | 伪造 JWT | mitigate | get_current_user 对每个请求验证签名和过期 |
| T-04-05 | Repudiation | 登录事件 | accept | 校园工具低风险，User.created_at 记录首次登录 |
</threat_model>
