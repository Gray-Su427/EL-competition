<action>
1. 修改 models.py 中 Canteen 模型，新增 3 个可空字段（per D-17，扩展客流数据存储）：
   - `current_people = Column(Integer, nullable=True)` — 当前人数（E-Mobile ttl 字段）
   - `occupancy_pct = Column(String, nullable=True)` — 拥挤度百分比（E-Mobile bl 字段，如 "65%"）
   - `flow_updated_at = Column(String, nullable=True)` — 最后更新时间 ISO 格式

2. 新建 `what_to_eat_today_web/backend/canteen_flow_service.py`（per D-14, D-15, D-16）：

   移植 canteen_flow.py 的核心逻辑，但用 httpx 替代 requests，适配后端异步架构：

   - 常量 `BASE_URL = "https://rll.nju.edu.cn/mobilemode/mobile"`, `SERVER_URL = f"{BASE_URL}/server.jsp"`
   - 常量 `META_INVOKER` 和 `NLIST_INVOKER`（与 canteen_flow.py 相同的 base64 值）
   - 常量 `SESSION_FILE_PATH`：从 .env 读取 `EMOBILE_SESSION_PATH`，默认为项目根目录 `session.json`
   - 常量 `REFRESH_INTERVAL_SECONDS = 600`（10 分钟）
   - 常量 `CANTEEN_NAME_MAP`：字典映射 E-Mobile canteenname → 数据库 canteen id，初始值 `{"鼓楼食堂一楼": "c1", "鼓楼食堂二楼": "c2", "鼓楼教工食堂": "c3"}`

   - 函数 `load_session() -> tuple[dict, str] | None`：读取 SESSION_FILE_PATH JSON 文件，返回 (cookies, session_key)，文件不存在或格式错误返回 None

   - 异步函数 `fetch_session_key(http_client: httpx.AsyncClient, cookies: dict) -> str | None`：GET server.jsp with action=meta params（复制 canteen_flow.py get_session_key 的 params），传入 cookies，解析 JSON 返回 data.sessionKey，异常返回 None

   - 异步函数 `fetch_canteen_data(http_client: httpx.AsyncClient, cookies: dict, session_key: str) -> list[dict] | None`：POST server.jsp with NListAction params + form_data（复制 canteen_flow.py fetch_canteen_data 的 params 和 form_data），传入 cookies，解析 JSON 提取 data.datas 列表，异常返回 None

   - 函数 `update_canteen_status(canteen_list: list[dict]) -> int`：同步函数，使用 SessionLocal，遍历 canteen_list，从 item.get("dataMap", item) 提取 canteenname/ttl/bl，通过 CANTEEN_NAME_MAP 映射到 canteen_id，session.query(Canteen).filter_by(id=canteen_id).update({...status 根据 bl 计算: bl<40% 为"空闲", 40-70% 为"正常", >70% 为"拥挤"...current_people=ttl, occupancy_pct=bl, flow_updated_at=now})，返回更新记录数

   - 异步函数 `refresh_canteen_flow(http_client: httpx.AsyncClient) -> None`：加载 session → fetch session_key（若已有则用已有，否则重新获取） → fetch_canteen_data → update_canteen_status，任何步骤失败打印警告日志并 return

   - 异步函数 `start_refresh_loop(http_client: httpx.AsyncClient) -> None`：无限循环，每次 refresh_canteen_flow 后 await asyncio.sleep(REFRESH_INTERVAL_SECONDS)，捕获 asyncio.CancelledError 退出，捕获 Exception 打印错误继续循环

3. 修改 .env 追加：`EMOBILE_SESSION_PATH=` （留空表示用默认路径）。同步 .env.example。
</action>

### Task 2: Lifespan 集成定时刷新任务

<read_first>
- what_to_eat_today_web/backend/main.py (当前 lifespan 结构)
- what_to_eat_today_web/backend/canteen_flow_service.py (start_refresh_loop 签名)
</read_first>

<action>
修改 `what_to_eat_today_web/backend/main.py` 的 lifespan 函数：

1. 在 startup 部分（yield 之前），在 `app.state.http_client` 创建之后：
   - `from canteen_flow_service import start_refresh_loop`
   - `import asyncio`
   - `app.state.flow_task = asyncio.create_task(start_refresh_loop(app.state.http_client))`
   - 打印 "Canteen flow refresh task started"

2. 在 shutdown 部分（yield 之后），在 http_client.aclose() 之前：
   - `app.state.flow_task.cancel()`
   - `try: await app.state.flow_task except asyncio.CancelledError: pass`
   - 打印 "Canteen flow refresh task stopped"

这样服务启动时自动开始定时刷新，关闭时优雅取消。若 session.json 不存在，start_refresh_loop 内部 load_session 返回 None，refresh_canteen_flow 打印警告并跳过，不影响服务正常运行。
</action>
